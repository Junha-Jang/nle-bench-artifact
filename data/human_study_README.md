# Human Study Records

`human_study.json` contains the anonymized human study ratings used for
the paper's inter-rater agreement and same-source human--metric
diagnostics (App. I, §3.5, §4.4). `human_study_per_row_provenance.csv`
provides the reviewer-facing per-row provenance table linking each
released row to its canonical/latest-use status without exposing real
evaluator names. The standalone `compute_iaa.py` helper recomputes the
released agreement statistics from these ratings; the paper-reported
L2 Spearman correlation additionally uses the study's model-output/SR
join and is reported in App. I as same-source rubric alignment rather
than independent validation.

## File

- **Path**: `data/human_study.json`
- **Format**: JSON array (1,796 records)
- **Size**: ~690 KB
- **Provenance CSV**: `data/human_study_per_row_provenance.csv` (1,086 canonical `(rater, scenario, layer)` tuples; includes 1,080 design tuples plus six re-rate/provenance tuples)

## Schema

Each record:

| Field | Type | Description |
|---|---|---|
| `id` | int | Internal record identifier (opaque, retained for de-duplication) |
| `layer` | str | One of `l1`, `l2`, `l3` |
| `rater` | str | Anonymized rater identifier: `annotator_a`–`annotator_f` |
| `scenario_id` | str | Scenario identifier (matches `code/src/nlebench/dataset/scenarios_v3_1/`) |
| `responses` | object | Layer-specific response payload (see below) |
| `created_at` | str | ISO 8601 UTC timestamp of first submission |
| `updated_at` | str | ISO 8601 UTC timestamp of last update (re-rate) |

### `responses` payload by layer

**L1 — Scenario Validation**
```json
{
  "realism": 1-5,
  "feasibility": "feasible" | "ambiguous" | "infeasible",
  "constraint_validity": "yes" | "no" | ...
}
```

**L2 — Human–Metric Correlation**
```json
{
  "model_a_score": 1-5,
  "model_b_score": 1-5
}
```

**L3 — Mixed-Expertise Decision Reference**
```json
{
  "decision": "execute" | "clarify" | "refuse",
  "reason": "<short free-text justification>",
  "tool_calls": [<list of tool-call objects>],
  "results": [<list of per-call success/error objects>]
}
```

## Counts

| Layer | Records | Unique (rater, scenario) tuples | Design |
|---|---:|---:|---|
| L1 | 1,107 | 604 | 200 scenarios × 3 raters = 600 (4 overlap re-rates) |
| L2 | 343 | 240 | 80 scenarios × 3 raters = 240 |
| L3 | 346 | 242 | 80 scenarios × 3 raters = 240 (2 overlap re-rates) |
| **Total** | **1,796** | **1,086** | **1,080 design** |

The L2 record count of 343 is a human-study record count, not the v3→v3.1
scenario-revision count. The released corpus has 344 unique YAMLs with
non-metadata content changes after excluding global annotator-metadata
normalization.

The ~6 surplus tuples reflect mid-study re-rates (same rater, same scenario, multiple submissions). For statistics, the latest `updated_at` per `(rater, scenario_id, layer)` is canonical.

## Anonymization

- Rater real identifiers replaced with `annotator_{a..f}` codes:
  - L1 raters (3 non-author professional video editors from a single convenience pool): `annotator_a`, `annotator_b`, `annotator_c`
  - L2/L3 raters (3 non-author mixed-expertise raters from the same coordinated annotator pool, same persons across both layers): `annotator_d`, `annotator_e`, `annotator_f`
- All six raters were non-author participants from a single convenience pool; the paper treats this as a closed-pool limitation rather than independent external validation.
- Pilot/sanity submissions by the principal investigator (11 records) are excluded from this release.
- L3 `reason` free-text fields are retained as-is; reviews of these for indirect identification surfaced no concerns.
- Timestamps preserved (no personally identifying information; required for re-rate handling).

## Reproducing IAA Statistics

A self-contained reproduction script is provided at `code/scripts/compute_iaa.py`. It computes:

- L1 Fleiss kappa (feasibility, 3-rated subset)
- L1 Krippendorff alpha (feasibility nominal; realism interval)
- Per-scenario realism mean and SD
- L2 Krippendorff alpha (full scenario-side design, model_a_score,
  model_b_score, A/B-averaged)
- L3 Fleiss kappa (decision, 3-rated subset)
- L3 average pairwise Cohen's kappa (decision, overlap subset)

```bash
cd code/scripts
python compute_iaa.py
```

The script depends only on the Python standard library (no scipy / numpy / pandas). It prints the full L2 interval alpha used in the
paper (`0.469`, rounded to `0.47`) plus component diagnostics; the
L2 Spearman `rho=0.55` remains a paper-reported same-source SR join,
not an independent external validation claim.

For each `(rater, scenario, layer)` tuple, the latest `updated_at` submission is canonical. Pilot/sanity rows are already excluded from the released JSON.

## Per-row Provenance CSV

`human_study_per_row_provenance.csv` is generated by `code/scripts/generate_review_artifacts.py`. It contains one row per canonical released `(rater, scenario_id, layer)` tuple, including the anonymized rater code, timestamps, a `response_match` flag, and a `source_evaluator_sha256` hash of the omitted source evaluator identifier. Real evaluator names are not released.

Cross-reference: paper Appendix I ("Human Study Protocol") for the full methodology and L1 IAA stratification result.
